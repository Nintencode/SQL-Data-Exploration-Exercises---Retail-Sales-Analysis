# SQL-Data-Exploration-Exercises---Retail-Sales-Analysis

This project demonstrates the use of PostgreSQL to clean, explore, and analyze retail sales data. The workflow begins by creating a database and table, importing a CSV dataset, validating the data, identifying missing values, and removing incomplete records to prepare the dataset for analysis.

The analysis answers a variety of business-focused questions using SQL, including identifying high-value transactions, calculating category-level sales, analyzing customer demographics, ranking the highest-performing sales months with window functions, finding top customers, and classifying sales into time-of-day shifts using CASE statements. The project also incorporates aggregate functions, filtering, grouping, sorting, subqueries, and window functions such as RANK(), providing hands-on practice with common SQL techniques used in data analysis.

This project was completed as a exercise based on a YouTube tutorial(please see link to the video in the description) and includes my own notes, comments, and modifications to reinforce the concepts and document the learning process.
